# 🧩 Overall Flow

```text
sortList(head)
   ↓
find middle
   ↓
split into 2 halves
   ↓
recursively sort both halves
   ↓
merge sorted lists
```

# 🧠 1. Main Function

```java
public ListNode sortList(ListNode head) {
    if (head == null || head.next == null) return head;
```

### 👉 Meaning:

- If list is empty OR only 1 node → already sorted ✅
- This is your **base case**

---

```java
    ListNode mid = getMid(head);
    ListNode rightHead = mid.next;
    mid.next = null;
```

### 👉 What’s happening here?

#### Step 1: Find middle

```
4 → 2 → 1 → 3
        ↑ mid
```

#### Step 2: Split list

```
Left:  4 → 2
Right: 1 → 3
```

👉 `mid.next = null` is **VERY IMPORTANT**

- It **breaks the list into two parts**
- Without this → infinite recursion ❌

---

```java
    ListNode left = sortList(head);
    ListNode right = sortList(rightHead);
```

### 👉 Recursion kicks in

Each half is sorted **independently**

Example:

```
4 → 2  → becomes → 2 → 4
1 → 3  → becomes → 1 → 3
```

---

```java
    return merge(left, right);
}
```

### 👉 Final step

Merge two sorted lists into one

# 🧠 2. Finding Middle (getMid)

```java
private ListNode getMid(ListNode head) {
    ListNode slow = head;
    ListNode fast = head.next;
```

### 👉 Why fast = head.next?

To **avoid infinite loop** and ensure proper split

---

```java
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
```

### 👉 Movement:

| Pointer | Speed   |
| ------- | ------- |
| slow    | 1 step  |
| fast    | 2 steps |

---

### 🔍 Example:

```
4 → 2 → 1 → 3
```

| Step  | slow    | fast |
| ----- | ------- | ---- |
| start | 4       | 2    |
| step1 | 2       | 3    |
| stop  | mid = 2 |      |

---

```java
    return slow;
}
```

👉 Returns **node BEFORE split point**

# 🧠 3. Merge Function (Most Important)

```java
ListNode dummy = new ListNode(-1);
ListNode tail = dummy;
```

### 👉 Why dummy node?

Avoids edge cases like:

- empty list
- first insertion

---

## 🔄 Main Merge Loop

```java
while (l1 != null && l2 != null) {
```

👉 Run until one list finishes

---

```java
    if (l1.val < l2.val) {
        tail.next = l1;
        l1 = l1.next;
    } else {
        tail.next = l2;
        l2 = l2.next;
    }
    tail = tail.next;
```

### 👉 What’s happening?

We:

1. Compare values
2. Attach smaller node
3. Move pointer forward

---

### 🔍 Example:

```
l1: 2 → 4
l2: 1 → 3
```

Step-by-step:

```
dummy → 1 → 2 → 3 → 4
```

---

## 🧹 Remaining Nodes

```java
if (l1 != null) tail.next = l1;
if (l2 != null) tail.next = l2;
```

👉 One list may still have elements left

---

```java
return dummy.next;
```

👉 Skip dummy → return actual head

# 🔁 Full Recursion Tree (Important)

```
4 → 2 → 1 → 3

Split:
[4,2]   [1,3]

Split:
[4] [2]   [1] [3]

Merge:
[2,4]   [1,3]

Final:
[1,2,3,4]
```

# ⚡ Key Concepts You Must Say in Interview

### ✅ 1. Why merge sort?

> “Linked lists don’t allow random access, so merge sort is optimal.”

---

### ✅ 2. Why slow-fast pointer?

> “To find middle in O(n) without extra space.”

---

### ✅ 3. Why dummy node?

> “Simplifies merging and avoids edge cases.”

# 🚨 Common Bugs (VERY IMPORTANT)

❌ Forgetting:

```java
mid.next = null;
```

❌ Doing:

```java
fast = head   // ❌ wrong split sometimes
```

❌ Losing references while merging
